Core Features
🗓️
Restart Scheduling
Any number of named schedules in schedules.yml — days, times, timezone, and priority. The earliest upcoming restart across all schedules always wins.
⚠️
Warnings & Countdown
Configurable countdown thresholds with chat messages, titles, sounds, and an action-bar countdown as the restart approaches.
🛡️
Grace Period
Delay a restart while players are in combat (via CombatLogX) or in a protected world, up to a configurable maximum delay.
📦
Pre-Restart Backups
Zips or copies configured world folders before restart, with maintenance-mode lockout and automatic pruning of old backups.
Performance Monitoring
Rolling TPS Window
TPS-Based Performance Triggers
Samples 1-minute TPS once per second into a rolling window. If every sample stays below the configured threshold for the full window, a restart is scheduled (or forced immediately) — with a cooldown to prevent restart loops.

Integration & Tooling
📣
Discord Notifications
Scheduled, manual, and server-online events relayed through the DiscordUtils plugin's webhook API — a soft-dependency, not a built-in webhook client.
🌐
Web Config Editor
A form-based YAML builder served through SwagAPI's shared panel. Exports config.yml / schedules.yml for manual install — no live server save yet.
🖥️
In-Game GUI
/srestart gui — browse and edit existing schedules, toggle warnings and backup settings, and view the last 10 restart log entries.
